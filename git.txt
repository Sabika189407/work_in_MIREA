Основные команды Git в Linux включают инициализацию (git init),
клонирование (git clone), добавление файлов (git add), создание фиксаций (git commit),
просмотр состояния (git status) и работу с удаленными репозиториями (git push/pull).
Эти команды обеспечивают базовый контроль версий, управление ветками (git branch, checkout)
и объединение и зменений (git merge).

1. Настройка и инициализация
Установка имени пользователя: git config --global user.name "Имя".
Установка email: git config --global user.email "email@example.com".
Инициализация репозитория: git init — создает новый репозиторий в текущей директории.
Клонирование репозитория: git clone <url> — копирует существующий репозиторий. 

2. Основной рабочий цикл (добавление и коммит)
Статус файлов: git status — показывает измененные файлы.
Добавление в индекс: git add <файл> или git add . (все файлы).
Фиксация изменений: git commit -m "Сообщение" — сохраняет снимок файлов. 

3. Работа с ветками
Список веток: git branch.
Создание ветки: git branch <имя_ветки>.
Переключение ветки: git checkout <имя_ветки> или git switch <имя_ветки>.
Создание и переход: git checkout -b <имя_ветки>.
Слияние веток: git merge <имя_ветки>. 

4. Удаленные репозитории (GitHub/GitLab)
Добавление удаленного репозитория: git remote add origin <url>.
Отправка изменений: git push -u origin <ветка>.
Получение изменений: git pull.
Скачивание без слияния: git fetch. 

5. Просмотр и отмена изменений
История коммитов: git log --oneline.
Разница файлов: git diff.
Отмена изменений файла: git checkout -- <файл>.
Сброс до коммита: git reset --hard <hash_коммита>.
Удаление файла: git rm <файл>. 

Установка Git в Linux
Debian/Ubuntu/Mint: sudo apt-get install git.
Fedora/RHEL/CentOS: sudo dnf install git.
Arch Linux: sudo pacman -S git.
